# Easy Slug Protect

## 概要

Easy Slug Protect は、WordPress サイトの特定の URL 階層（スラッグ）に対して、シンプルなパスワード保護機能を提供するプラグインです。

指定したパスとその配下のページや投稿へのアクセスを制限し、設定されたパスワードを知っているユーザーのみがコンテンツを閲覧できるようになります。

## 主な機能

- **パス保護**: 特定の URL パス（例: `/members/`, `/private-docs/`）を指定して、そのパス以下のコンテンツを保護します。
- **パスワード認証**: 保護されたパスごとに個別のパスワードを設定できます。
- **専用ログインページ**: 作成した固定ページにショートコード `[esp_login_form]` を設置することで、パスワード入力フォームとして利用できます。
- **ブルートフォース対策**: 一定回数以上ログインに失敗した IP アドレスからのアクセスを一時的にブロックし、不正アクセス試行を防ぎます。
- **ログイン保持**: 「ログイン状態を記憶する」オプションにより、ユーザーは設定された期間ログイン状態を維持できます。
- **メール通知**: 新しい保護パスの追加、パスワードの変更、保護パスの削除、重大なエラーの発生といったイベントが発生した際に、サイト管理者へメールで通知します。
- **コンテンツフィルタリング**: ログインしていないユーザーに対しては、保護されたコンテンツを検索結果やアーカイブページから除外します。
- **管理ツール**:
  - **パーマリンクパス情報再生成**: 投稿のパーマリンクとプラグイン内部のパス情報との間に不整合が生じた場合に、全投稿のパス情報を再生成します。
  - **保護キャッシュクリア**: 保護対象の投稿リストのキャッシュを強制的にクリアします。

## ショートコード

### ログインフォーム: `[esp_login_form]`

保護されたコンテンツへのアクセスに必要なパスワード入力フォームを表示します。通常、ログインページとして指定した固定ページに設置します。

- `path=""`: (オプション) ログインページとして設定されていないページにフォームを設置する場合、どの保護パス用のフォームかを指定します。例: `[esp_login_form path="/protected-area/"]`
- `place_holder=""`: (オプション) パスワード入力欄のプレースホルダーテキストを指定します。

### ログアウトボタン: `[esp_logout_button]`

保護されたエリアからログアウトするためのボタンを表示します。

- `text=""`: (推奨) ボタンに表示されるテキストを指定します。例: `[esp_logout_button text="ログアウトする"]`
- `redirect_to=""`: (オプション) ログアウト後にリダイレクトする先の URL を指定します。未指定の場合はサイトのホームページにリダイレクトされます。
- `class=""`: (オプション) ボタンに任意の CSS クラスを追加します。
- `path=""`: (オプション) 複数の保護パスを設定している場合、特定のパスからのみログアウトさせたい場合にそのパスを指定します。未指定の場合、ボタンが設置されているページのパスに基づいてログアウト処理が行われます。
